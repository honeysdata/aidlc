# 테이블오더 서비스 페르소나

## Persona 1: 고객 (Customer)

### 기본 정보
- **역할**: 매장 방문 고객
- **사용 환경**: 테이블에 설치된 태블릿 (웹 브라우저)
- **사용 빈도**: 매장 방문 시마다 (일회성~정기적)

### 목표
- 빠르고 편리하게 음식 주문하기
- 대기 시간 없이 원하는 메뉴 선택하기
- 주문 상태를 실시간으로 확인하기

### 니즈
- 직관적이고 사용하기 쉬운 인터페이스
- 메뉴 정보(이름, 가격, 설명)를 명확하게 확인
- 주문 전 장바구니에서 수량 조절 및 확인
- 주문 후 진행 상태 파악

### 페인포인트
- 복잡한 로그인 절차
- 느린 페이지 로딩
- 주문 실패 시 불명확한 에러 메시지
- 이전 주문과 현재 주문의 혼동

### 기대사항
- 별도 로그인 없이 바로 주문 가능
- 터치 친화적인 큰 버튼
- 주문 성공/실패에 대한 명확한 피드백
- 현재 세션의 주문만 깔끔하게 표시

---

## Persona 2: 매장 관리자 (Admin)

### 기본 정보
- **역할**: 매장 운영자/관리자
- **사용 환경**: PC 또는 태블릿 (웹 브라우저)
- **사용 빈도**: 매일 영업 시간 동안 상시

### 목표
- 들어오는 주문을 실시간으로 모니터링
- 테이블별 주문 상태를 효율적으로 관리
- 메뉴와 카테고리를 유연하게 관리

### 니즈
- 신규 주문 즉시 알림 (시각적 강조)
- 테이블별 주문 현황 한눈에 파악
- 주문 상태 빠르게 변경 (대기중→준비중→완료)
- 메뉴/카테고리 동적 추가/수정/삭제

### 페인포인트
- 주문 누락으로 인한 고객 불만
- 테이블 세션 관리의 복잡함
- 잘못된 주문 수정의 어려움
- 과거 주문 내역 조회의 불편함

### 기대사항
- 2초 이내 실시간 주문 반영
- 직관적인 대시보드 레이아웃
- 주문 삭제 및 테이블 이용 완료 처리 기능
- 날짜별 과거 주문 내역 조회

## 페르소나-스토리 매핑

| 페르소나 | 주요 사용자 여정 |
|---------|-----------------|
| 고객 | 주문 플로우 (로그인→메뉴조회→장바구니→주문→내역확인) |
| 관리자 | 관리 플로우 (로그인→모니터링→상태변경→테이블관리→메뉴관리) |
